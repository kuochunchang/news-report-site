3. David Fowler 為 AI 代理發布 dotnet-skillz

Category: Open Source Medium

概要： Microsoft 的 David Fowler 發布了「dotnet-skillz」，這是一個 CLI 工具，可將 .NET DLL 反編譯為 AI 代理可讀的 C# 程式碼。這解決了 .NET 代理與 JavaScript 代理相比所面臨的劣勢，後者可以輕鬆讀取原始碼分發。

背景： AI 代理在處理 .NET 時經常遇到困難，因為該生態系統分發的是二進位檔案而非原始原始碼。

影響分析： 顯著提升了 AI 編碼代理在 .NET 生態系統中的能力，使其能夠更好地分析遺留程式碼庫。

5. 從 MCP 轉向基於 CLI 的代理技能

Category: Open Source Medium

概要： 開發者之間日益增長的趨勢顯示，他們正從模型上下文協定 (MCP) 轉向更精簡的基於 CLI 的「技能」。這一轉變是由於需要降低 Token 成本並防止代理上下文膨脹。

背景： MCP 最初被設計為代理與數據互動的標準化方式，但開發者發現它對於簡單任務來說越來越「笨重」。

關鍵觀點：

  • MCP 是一個錯誤；代理最好是編寫 bash 腳本或使用直接的 API 端點。—— @melvynxdev (MCP is a mistake; agents are better off writing bash scripts or using direct API endpoints - @melvynxdev)

  • 代理更喜歡 CLI 技能，因為它們可以寫入文件並迭代使用 grep 以節省 Token。—— @davidfowl (Agents prefer CLI skills because they can write to files and grep iteratively to save tokens - @davidfowl)

影響分析： 隨著開發者將效率置於通用協定之上，這可能導致代理通訊標準的碎片化。
