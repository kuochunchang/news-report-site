**關鍵觀點：**

### 10. AI Agent 安全漏洞：OpenClaw 零點擊漏洞與 Claude Code 缺陷引發業界警戒

| 屬性 | 值 |
|------|------|
| **分類** | Policy |
| **熱度** | Medium |

**概要：** 2026 年 3 月初，AI Agent 框架中的關鍵安全漏洞已成為主要關注焦點。OpenClaw 零點擊漏洞允許惡意網站透過本地端 WebSocket 暴露遠端劫持開發者的 AI Agent，無需進行身份驗證——使攻擊者能夠從整合服務（如 Slack）中擷取 API 金鑰、竊取檔案，並在連接的節點上執行 Shell 命令，包括 macOS 應用程式和 iOS 裝置。同時，Claude Code 也被標記存在遠端程式碼執行（RCE）漏洞，以及 CLI 工具抹除使用者資料的報告。這些事件引發了網路安全專家的緊急呼籲，要求在隔離的 VM 或容器中對 AI Agent 進行沙箱化、為機器人實施零信任架構，並為本地端防禦採用 mTLS 與工具允許清單及審計日誌。

**背景：** AI Agent 框架迅速普及，組織部署能夠執行命令、存取 API 以及在開發者機器上操作檔案的自主系統。OpenClaw 框架旨在跨多個節點協調 AI Agent，其在本地端暴露了一個缺乏身份驗證的 WebSocket 閘道。這一漏洞代表了 AI 系統設計中的根本缺陷——同時授予無限制的系統存取權限、維持外部通訊通道，並處理不受信任的輸入——即專家所稱的「AI 安全三角」。隨著 AI Agent 越來越多地自主處理敏感操作（如軟體購買和資料移動），攻擊面已大幅擴展。這一時間點與企業快速採用 Agentic AI 工作流程相吻合，使得這些漏洞對組織的安全態勢具有特別重大的影響。

**關鍵觀點：**

- 「AI 安全三角」——無限制存取加上外部通訊加上不受信任的輸入——造成了可擴展的污染風險。資料層面的修復，包括來源追蹤和人類驗證，是讓 AI 根植於可驗證現實的必要條件。解決方案需要根本的架構變革，而非表面的修補。（@PerleLabs）

- 能夠自主購買軟體和移動資料的 Agentic AI 到 2026 年底將成為企業頭號攻擊向量。組織必須為機器人實施零信任，並進行 AI 威脅行為者桌面演練，以準備面對這一新現實。（@mclynd）

- 在 VM 或容器中執行 Agent，而非在主要的開發者機器上，對於透過不受信任檔案進行提示注入和無狀態會話漏洞的緩解至關重要。這種隔離方法應該是任何生產環境 AI Agent 部署的強制性要求。（@loxhard1205）

- 本地端防禦需要 mTLS 身份驗證、限制 Agent 可執行命令的工具允許清單，以及追蹤所有 Agent 動作的全面審計日誌。將本地端視為可信賴的假設在 Agentic 系統中根本上是錯誤的。（@EUAIACTGUY）

- AI Agent 大幅擴展了企業環境中的 breach 點。每個具有系統存取權限的 Agent 代表著潛在的橫向移動向量，攻擊者在獲得初始存取權限後可加以利用。（@keeganitreal）

**影響分析：** 這些漏洞的立即影響對於已在工作流程中部署 AI Agent 的開發者和組織而言意義重大。OpenClaw 零點擊漏洞僅需開發者訪問惡意網站，就可能危及其整個開發環境，包括 API 金鑰、原始碼和系統層級存取的暴露。在短期內，組織應立即審計其 AI Agent 部署、在可能的情況下停用 WebSocket 暴露，並實施網路隔離。長期而言，這些漏洞加速了對 AI Agent 框架行業範圍安全標準的需求，可能影響類似於歐盟 AI Act 下正在開發的監管框架。該事件也可能會減緩企業對 Agentic AI 的採用，直到開發出更強健的安全架構，同時推動對沙箱化、預設安全 Agent 解決方案的需求，使其能夠在不使用主機系統處理不受信任輸入的情況下運作。
