2. Google Cloud 採用 MCP 作為多代理系統標準

Category: 產業 (Industry) 高

概要： Google Cloud 已正式整合模型上下文協定 (Model Context Protocol, MCP)，這是最初由 Anthropic 提出的開放標準，旨在讓 AI 代理能夠安全地連接到外部工具和資料庫。該協定被描述為「AI 整合界的 USB-C」，允許模型與 BigQuery、Google Kubernetes Engine (GKE) 和 Gmail 等服務之間進行運行時發現和雙向通訊。Google Cloud Tech 最近推出了用於編排多代理系統的新實驗室，其中 MCP 與代理對代理 (A2A) 協定共同充當「任務路由器」。雖然其可擴展性受到讚譽，但開發者報告在設置 Google Workspace 身分驗證以及在生產環境中調試 MCP 伺服器時存在摩擦。儘管有這些障礙，生態系統仍在迅速增長，各雲端供應商的 MCP 伺服器下載量已超過 800 萬次。

背景： MCP 的創建是為了編解決「N×M」問題，即以前每個新 AI 模型都需要為每個新工具進行自定義整合。透過介面標準化，任何符合 MCP 標準的模型都可以立即使用任何符合 MCP 標準的工具，顯著降低了構建複雜代理工作流的門檻。

  • MCP 「一次構建，隨處使用」的理念對於與大型團隊競爭的獨立創始人來說是一個遊戲規則改變者 — @johanrin

5. AgentHQ 發布：用於編碼代理的遠端終端多路復用器

Category: 產品發布 (Product Launch) 中

概要： NVIDIA AI 研究員 Rick Lamers 發布了「agenthq」，這是一個開源的基於終端的「編碼代理多路復用器 (multiplexer)」。該工具僅用兩天時間建成，允許開發者在高性能遠端基礎設施上運行重型 AI 編碼代理（如 Claude Code），同時從任何設備（包括手機）進行控制。主要功能包括透過守護進程 (daemon) 串流 stdin/stdout、自動 git 工作樹處理以及原生的代理式代碼合併。該項目受到了 NVIDIA 研究社群和需要持久遠端代理會話的獨立開發者的熱烈歡迎。Lamers 指出，該工具受到 Conductor 和 Supacode 等其他項目的啟發，但專注於自託管、採用 MIT 許可證的技術棧。

背景： 編碼代理通常需要大量的本地算力或特定的環境才能有效運行。AgentHQ 解決了「持久性」問題，允許開發者在桌面上啟動編碼任務，並在代理於遠端伺服器工作時透過行動終端進行監控和互動。
